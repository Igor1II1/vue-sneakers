# GitFlow — Vue Sneakers

В проекте используется стратегия ветвления на основе GitFlow, адаптированная для небольшой команды.

## Основные ветки

**main** — стабильная версия, готовая к деплою. Все изменения попадают в main только через Pull Request. Напрямую в main не коммитим.

**develop** — основная ветка разработки. Сюда вливаются все новые функции и исправления после ревью. Из develop периодически создаётся PR в main для релиза.

**task** — ветка с документацией проекта (task.md, terms.md, review.md и другие описательные файлы).


## Вспомогательные ветки

**feature/*** — для новой функциональности. Создаётся от develop, после завершения вливается обратно через Pull Request.

Примеры:
- `feature/mobile-layout` — адаптивная вёрстка
- `feature/product-page` — страница отдельного товара
- `feature/brand-filter` — фильтр по бренду

**bugfix/*** — для исправления ошибок в процессе разработки. Создаётся от develop.

Примеры:
- `bugfix/cart-save-error` — ошибка сохранения корзины
- `bugfix/search-clear` — поиск не сбрасывается

**hotfix/*** — для срочных исправлений в продакшене. Создаётся от main, после исправления вливается и в main, и в develop.

Примеры:
- `hotfix/broken-images` — не загружаются картинки на проде
- `hotfix/drawer-crash` — корзина не открывается


## Правила именования веток

- Нижний регистр, слова через дефис
- Кратко и по сути: `feature/favorites-page`, не `feature/new-page-for-favorites-list-with-items`
- Без транслита: используем английский


## Процесс работы

1. Взять задачу из списка (Issues или устная договорённость)
2. Создать ветку от develop: `git checkout -b feature/название develop`
3. Работать, делать промежуточные коммиты
4. Запушить ветку: `git push -u origin feature/название`
5. Открыть Pull Request в develop
6. Пройти ревью (или самопроверку, если работаешь один)
7. Смержить и удалить ветку
8. Раз в спринт — PR из develop в main для релиза


## Правила коммитов

Сообщения пишем на русском или английском — главное единообразие в рамках одной ветки.

Используем префиксы:
- `feat:` — новая функциональность (`feat: добавить фильтр по цене`)
- `fix:` — исправление бага (`fix: корзина не сохраняется после перезагрузки`)
- `style:` — изменения в стилях, форматировании (`style: выровнять карточки на мобильных`)
- `refactor:` — рефакторинг без изменения поведения (`refactor: вынести логику корзины в composable`)
- `docs:` — документация (`docs: обновить README`)
- `chore:` — мелкие задачи, настройка инструментов (`chore: обновить зависимости`)


## Тегирование версий

При каждом релизе в main создаётся тег:
- `v1.0.0` — первый релиз (MVP)
- `v1.1.0` — новые функции
- `v1.1.1` — багфиксы

Формат: `v<major>.<minor>.<patch>` (семантическое версионирование).
