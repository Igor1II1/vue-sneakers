# Оптимизация — Vue Sneakers

## 1. Для чего нужна оптимизация

- Ускорить загрузку страницы — чем быстрее сайт, тем больше покупок
- Улучшить позиции в поисковиках — скорость напрямую влияет на ранжирование
- Повысить удобство на мобильных устройствах — больше половины пользователей заходят с телефонов
- Снизить процент отказов — пользователь уходит, если страница грузится дольше 3 секунд
- Подготовить проект к масштабированию — при увеличении каталога сайт не должен тормозить


## 2. Что будем оптимизировать

### 2.1. Изображения

Текущее состояние: все картинки кроссовок в формате JPG, без сжатия, загружаются сразу все 12 штук.

Проблемы:
- JPG тяжелее, чем WebP/AVIF
- Все картинки загружаются при открытии страницы, даже если пользователь их не видит
- Нет указания размеров (width/height), что вызывает сдвиг макета при загрузке

### 2.2. Производительность JavaScript

Текущее состояние: бандл включает весь код приложения в одном файле.

Проблемы:
- Страница Favorites загружается в бандл, даже если пользователь на неё не заходит
- lodash.debounce подтягивает лишний код, хотя нужна одна функция

### 2.3. Мобильная адаптация

Текущее состояние: сетка товаров фиксированная — 4 колонки.

Проблемы:
- На экранах меньше 768px карточки сжимаются до нечитаемого размера
- Drawer занимает 384px — на мобильном это весь экран
- Шапка не адаптируется под узкие экраны

### 2.4. SEO

Текущее состояние: SPA без серверного рендеринга.

Проблемы:
- Поисковые боты не видят контент, который загружается через JavaScript
- Нет мета-тегов (title, description, og:image)
- Нет sitemap.xml и robots.txt


## 3. Как будем оптимизировать

### 3.1. Изображения

| Действие | Как реализовать |
|----------|----------------|
| Конвертация в WebP | Перегнать все JPG в WebP через Squoosh или CLI-утилиты |
| Lazy loading | Добавить `loading="lazy"` на теги `<img>` в карточках |
| Указание размеров | Добавить атрибуты `width` и `height` ко всем изображениям |
| Адаптивные размеры | Использовать `srcset` для разных размеров экрана |

### 3.2. Производительность JavaScript

| Действие | Как реализовать |
|----------|----------------|
| Code splitting | Настроить ленивую загрузку маршрутов: `() => import('./pages/Favorites.vue')` |
| Tree shaking | Vite делает это автоматически при сборке |
| Минификация | Уже включена в Vite по умолчанию (esbuild) |
| Кэширование | Настроить заголовки кэширования для статики при деплое |

### 3.3. Мобильная адаптация

| Действие | Как реализовать |
|----------|----------------|
| Адаптивная сетка | Заменить `grid-cols-4` на `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` |
| Адаптивный Drawer | На мобильных — полная ширина, `w-full sm:w-96` |
| Адаптивная шапка | Перенести навигацию в бургер-меню на мобильных |
| Контейнер | Заменить `w-4/5` на `w-full sm:w-4/5` с добавлением `px-4` |

### 3.4. SEO

| Действие | Как реализовать |
|----------|----------------|
| Мета-теги | Добавить title и description в index.html |
| Open Graph | Добавить og:title, og:description, og:image |
| robots.txt | Создать файл в public/ |
| sitemap.xml | Сгенерировать для основных страниц |
| Prerendering | Использовать vite-plugin-prerender для статических страниц |


## 4. Сравнительные результаты

Ожидаемые метрики до и после оптимизации:

| Метрика | До оптимизации | После оптимизации |
|---------|---------------|-------------------|
| LCP (Largest Contentful Paint) | ~1.8 с | < 1.0 с |
| FCP (First Contentful Paint) | ~0.8 с | < 0.5 с |
| CLS (Cumulative Layout Shift) | ~0.15 | < 0.05 |
| Размер бандла JS | ~180 KB | < 120 KB |
| Размер изображений | ~1.2 MB (12 JPG) | < 400 KB (12 WebP) |
| Lighthouse Performance | ~75 | > 95 |
| Мобильный Lighthouse | ~45 | > 85 |
| Индексация поисковиками | Не индексируется | Основные страницы индексируются |
