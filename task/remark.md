# Обсуждение для развития и улучшений — Vue Sneakers

## Что работает хорошо

### Архитектура
Компонентная структура выстроена логично. Каждый компонент отвечает за одну задачу: Card — отображение товара, Drawer — корзина, Header — навигация. Это облегчает поддержку и расширение.

### Управление состоянием
Корзина управляется через provide/inject, что для проекта такого размера — правильное решение. Нет лишней сложности в виде Vuex или Pinia. При этом данные реактивные, за ними следят watch'еры, корзина автоматически сохраняется в localStorage.

### Скорость
Vite + статический JSON дают практически мгновенную загрузку. Нет ожидания ответа от сервера, нет спиннеров. Пользователь видит товары сразу.

### Визуальная часть
Tailwind CSS позволил сделать аккуратный интерфейс без написания CSS-файлов. Hover-эффекты на карточках, плавные анимации через auto-animate — всё это создаёт приятное впечатление.


## Что можно улучшить

### Адаптивность — приоритет номер один
Сейчас сайт выглядит хорошо только на экранах от 1024px. На телефоне карточки сжимаются, текст становится нечитаемым, шапка ломается. Учитывая, что больше 60% трафика в e-commerce приходится с мобильных — это критично.

Предложение: добавить responsive-классы Tailwind (`sm:`, `md:`, `lg:`) в сетку, шапку и Drawer.

### Страница товара
Сейчас нет возможности посмотреть товар подробнее — нет отдельной страницы с большим фото, описанием и выбором размера. Для магазина кроссовок это один из ключевых экранов.

Предложение: добавить маршрут `/product/:id` с компонентом ProductPage.vue.

### Обработка ошибок
Если JSON не загрузился или localStorage повреждён — пользователь увидит пустую страницу без объяснения. Нет обработки граничных случаев.

Предложение: добавить компонент-заглушку с сообщением «Не удалось загрузить товары» и кнопкой «Попробовать снова».

### Состояние загрузки
При первой загрузке каталога нет индикатора загрузки. Пользователь на долю секунды видит пустую страницу.

Предложение: добавить skeleton-заглушки карточек на время загрузки данных.

### Валидация заказа
Кнопка «Оформить заказ» работает, но нет формы с контактными данными. Заказ сохраняется без имени, телефона и адреса.

Предложение: перед оформлением заказа показывать форму с обязательными полями.

### Масштабируемость каталога
12 товаров загружаются из одного JSON-файла целиком. При увеличении каталога до сотен позиций это перестанет работать эффективно.

Предложение: при масштабировании реализовать пагинацию или бесконечный скролл, перенести данные на бэкенд.
